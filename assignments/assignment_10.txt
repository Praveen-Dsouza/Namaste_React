2) How do we configure tailwind?
Install Tailwind CSS
- npm install -D tailwindcss postcss
- npx tailwindcss init

Config PostCSS
create file .postcss, add the below code into it.
{
  "plugins": {
    "tailwindcss": {}
  }
}

Configure you template path:
A tailwind config file will be created when we ran the command: npx tailwindcss init
Add the configuration into the file,

/** @type {import('tailwindcss').Config} */
module.exports = {
  content: [
    "./src/**/*.{html,js,ts,jsx,tsx}",
  ],
  theme: {
    extend: {},
  },
  plugins: [],
}

Add Tailwind directives to your CSS in index.css
@tailwind base;
@tailwind components;
@tailwind utilities;

3) In tailwind.config.js, what does all the keys mean (content, theme, extend, plugins)?
- content: The content array defines what files tailwind can be applied.
- theme: Where you define your color palette, fonts, type scale, border sizes, breakpoints â€” anything related to the visual design of your site.
- extend: To override the default theme.
- plugins: Allows you to register plugins with Tailwind that can be used to generate extra utilities, components, base styles, or custom variants.
